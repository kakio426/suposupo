# Week 1 Student Journeys

Date: 2026-05-10

## Journey 1: First Visit

| Item | Detail |
| --- | --- |
| Starting point | Student lands on `/` with no meaningful app context. |
| Student expectation | Understand what to tap first without knowing curriculum or grade level. |
| Current success state | `Lv.1부터 시작하기` and `1분 진단으로 맞춤 시작` are visible on the first screen. |
| Drop-off risks | Two start CTAs compete; save state appears before the student has experienced value; dense world catalog appears later on the same page. |
| Redesign direction | Make the first screen answer one question: “What should I do now?” Keep world depth below the fold or behind a world map entry. |

## Journey 2: Placement Start

| Item | Detail |
| --- | --- |
| Starting point | Student sees `1분 진단으로 맞춤 시작` from home. |
| Student expectation | The app finds the right level without exposing grade labels or creating test anxiety. |
| Current success state | Placement exists, uses 8 mixed addition questions, and maps to Lv.1, Lv.6, Lv.11, or Lv.16. |
| Drop-off risks | The term `진단` can feel evaluative; the flow does not preview how short/safe the placement will feel; placement result is not visually separated from normal result style enough. |
| Redesign direction | Rename and frame placement as a friendly path-finder. Add a short pre-start reassurance and result summary that explains “맞춤 시작점” rather than “score.” |

## Journey 3: Level Continue

| Item | Detail |
| --- | --- |
| Starting point | Returning student opens home with local or synced progress. |
| Student expectation | Resume the exact next useful challenge quickly. |
| Current success state | Home shows current level, opened level, completion count, and recent attempt. |
| Drop-off risks | Home has too many secondary sections before the student builds a habit; current world and exact next action could be visually stronger. |
| Redesign direction | Create a focused “Today” module with one primary action, one progress cue, and one small reason why this level is next. |

## Journey 4: Quiz Complete

| Item | Detail |
| --- | --- |
| Starting point | Student taps start and enters a 5-question quiz. |
| Student expectation | Quickly answer with touch, receive understandable feedback, and know whether to continue. |
| Current success state | Large 2x2 answer grid, clear progress indicator, immediate feedback, automatic next question, result screen after 5 questions. |
| Drop-off risks | Auto-advance may be too quick for students who need to inspect an error; feedback does not yet teach why the answer was wrong; current question card is symbol-heavy. |
| Redesign direction | Add optional “tap to continue” after wrong answers, richer visual manipulatives by stage, and concise repair hints tied to `mistakeType`. |

## Journey 5: Failure And Review

| Item | Detail |
| --- | --- |
| Starting point | Student finishes below the advance threshold. |
| Student expectation | Avoid shame and know the next small action. |
| Current success state | Result tone stays positive and recommends retry. |
| Drop-off risks | The app still shows score prominently; review path is not specific to the weak concept; no reward or progress preservation is shown after failure. |
| Redesign direction | Reframe failure as “repair mode.” Show one weak skill, one tiny next step, and a non-punitive energy/retry metaphor. |
